# 《交通事件文本结构化识别系统》标注规范

## 1. 标注目标

本规范用于指导项目团队对交通事件文本进行人工标注，形成可用于规则优化、模型训练、效果评估和系统测试的标准数据集。

标注对象包括事故警情、道路巡查记录、群众投诉、交通新闻和舆情文本等中文交通事件文本。

## 2. 标注任务

本项目标注任务分为三类：

1. 事件类型标注；
2. 实体字段标注；
3. 事件关系标注。

MVP 阶段优先完成事件类型标注和实体字段标注，关系标注可作为后续扩展。

## 3. 事件类型标注规范

### 3.1 一级事件类型

| 类型 | 编码 | 判定依据 |
|---|---|---|
| 交通事故 | ACCIDENT | 文本中出现追尾、碰撞、剐蹭、侧翻、撞人等事故描述 |
| 道路拥堵 | CONGESTION | 文本主要描述拥堵、通行缓慢、车辆排队等情况 |
| 道路施工 | CONSTRUCTION | 文本主要描述施工、养护、抢修、占道作业 |
| 交通管制 | CONTROL | 文本主要描述封闭、限行、分流、禁止通行等管制措施 |
| 设施故障 | FACILITY_FAULT | 文本主要描述信号灯、井盖、护栏、标志牌等设施异常 |
| 恶劣天气影响 | WEATHER | 文本主要描述暴雨、大雾、积水、结冰等天气影响 |
| 违法行为 | VIOLATION | 文本主要描述违停、逆行、占道、闯红灯等违法行为 |
| 其他事件 | OTHER | 无法归入上述类别 |

### 3.2 主事件类型判定原则

1. 如果文本中包含事故并导致拥堵，主类型标注为“交通事故”，拥堵内容标注为道路影响。
2. 如果文本中施工同时引发管制，主类型标注为“道路施工”，管制内容标注为道路影响或处置措施。
3. 如果文本主要描述管制通知，主类型标注为“交通管制”。
4. 如果文本仅描述车流缓慢、车辆排队，不出现事故、施工、故障等原因，主类型标注为“道路拥堵”。
5. 如果文本包含多个并列事件，可标注主事件类型，并在备注或扩展字段中记录次要事件。

## 4. 实体字段标注规范

### 4.1 实体类别

| 实体类别 | 标注内容 | 示例 |
|---|---|---|
| TIME | 事件发生时间 | 今天上午8点30分、昨日下午5点、早高峰 |
| LOCATION | 完整地点或区域 | 重庆市南岸区学府大道、人民路与建设路路口 |
| ROAD | 道路、桥梁、隧道、路口名称 | 学府大道、嘉陵江大桥、解放碑隧道 |
| DIRECTION | 道路方向或行驶方向 | 进城方向、由南向北、双向 |
| VEHICLE | 车辆类型或车辆数量 | 小轿车、电动车、两车、货车 |
| CASUALTY | 伤亡情况 | 1人轻伤、2人受伤、无人员伤亡 |
| ROAD_IMPACT | 道路影响 | 占用右侧车道、车辆排队约500米、禁止通行 |
| CONGESTION | 拥堵程度表达 | 通行缓慢、交通拥堵、严重拥堵 |
| DISPOSAL | 处置措施 | 交警已到场处理、已设置警示标志、建议绕行 |
| CAUSE | 事件原因 | 暴雨导致、信号灯故障、车辆追尾 |

### 4.2 标注边界规则

1. 时间实体应完整标注原文中的时间表达，如“今天上午8点30分”，不要只标“8点30分”。
2. 地点实体应优先标注完整地点，如“重庆市南岸区学府大道”，不要只标“学府大道”。
3. 道路实体可单独标注道路名称，如“学府大道”。
4. 车辆实体应标注车辆类型或数量表达，如“小轿车”“电动车”“两车”。
5. 伤亡实体应完整标注人数和伤亡程度，如“1人轻伤”“无人员伤亡”。
6. 道路影响实体应包含影响动作和对象，如“占用右侧车道”“车辆排队约500米”。
7. 处置措施实体应包含执行动作，如“交警已到场疏导”“巡查人员已设置警示标志”。

## 5. 事件关系标注规范

关系标注用于描述事件与实体之间的语义关系，适合后续关系抽取模型或事件知识图谱扩展。

| 关系类型 | 含义 | 示例 |
|---|---|---|
| 发生时间 | 事件发生于某时间 | 追尾事故 - 发生时间 - 今天上午8点30分 |
| 发生地点 | 事件发生于某地点 | 追尾事故 - 发生地点 - 学府大道 |
| 涉及车辆 | 事件涉及某车辆 | 碰撞事故 - 涉及车辆 - 小轿车 |
| 造成后果 | 事件造成伤亡或影响 | 追尾事故 - 造成后果 - 1人轻伤 |
| 影响道路 | 事件影响道路通行 | 追尾事故 - 影响道路 - 占用右侧车道 |
| 处置措施 | 事件对应处置动作 | 信号灯故障 - 处置措施 - 交警已到场疏导 |
| 发生原因 | 事件由某因素导致 | 道路拥堵 - 发生原因 - 暴雨积水 |

## 7. 标注质量控制

建议采用“双人标注 + 交叉复核”的方式控制质量。

流程如下：

1. 项目组先制定统一标注说明；
2. 两名成员独立标注同一批样本；
3. 对标注不一致的样本进行讨论；
4. 由第三名成员或组长确认最终标注；
5. 形成标准答案数据集；
6. 将修订后的规则同步到本规范。

## 8. 常见歧义处理

| 情况 | 处理方式 |
|---|---|
| “事故导致拥堵” | 主类型标注为交通事故，拥堵标注为道路影响 |
| “施工实施管制” | 主类型标注为道路施工，管制标注为道路影响或处置措施 |
| “群众反映长期拥堵” | 时间可为空，事件类型标注为道路拥堵 |
| “未造成人员伤亡” | casualty 标注为“无人员伤亡”，injured_count 和 death_count 均为 0 |
| “两车追尾” | vehicle_count 标注为 2，vehicles_involved 可为空 |
| “小轿车与电动车碰撞” | vehicles_involved 标注为小轿车、电动车，vehicle_count 标注为 2 |
