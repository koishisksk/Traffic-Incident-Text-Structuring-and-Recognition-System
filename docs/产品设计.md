# 《交通事件文本结构化识别系统》产品设计

## 1. 产品定位

《交通事件文本结构化识别系统》面向交通管理、道路巡查、城市治理和交通数据分析场景，目标是将事故警情、道路巡查记录、群众投诉、交通新闻和舆情文本等非结构化交通文本，自动转换为标准化交通事件数据。

系统重点解决交通文本“来源多、格式不统一、人工整理效率低、后续统计困难”的问题，为交通事件管理、数据统计、态势分析和可视化展示提供基础数据支撑。

本项目定位为大学生创新创业训练项目原型系统，优先实现可运行、可展示、可扩展的最小可行版本，后续逐步引入 NLP 模型和大语言模型辅助校验能力。

## 2. 用户对象

| 用户类型 | 使用需求 |
|---|---|
| 交通管理部门工作人员 | 快速整理警情、巡查、投诉等文本，形成结构化事件台账 |
| 道路巡查人员 | 将巡查记录转化为标准字段，便于后续查询和统计 |
| 数据分析人员 | 基于结构化数据分析高发路段、高发时段和事件类型分布 |
| 项目团队成员 | 进行规则开发、模型训练、数据标注和系统演示 |
| 竞赛/答辩评审 | 通过系统演示理解项目价值和技术路线 |

## 3. 核心使用场景

### 3.1 单条文本识别

用户输入一段交通事件文本，系统自动识别事件类型、时间、地点、车辆、伤亡、道路影响、拥堵程度等字段，并输出 JSON 结构化结果。

示例输入：

```text
今天上午8点30分，重庆市南岸区学府大道发生两车追尾事故，造成1人轻伤，占用右侧车道，现场通行缓慢。
```

示例输出：

```json
{
  "event_type": "交通事故",
  "event_subtype": "追尾事故",
  "event_time": "2026-07-07T08:30:00",
  "location": "重庆市南岸区学府大道",
  "vehicle_count": 2,
  "casualties": "1人轻伤",
  "road_impact": "占用右侧车道；通行缓慢",
  "congestion_level": "缓慢"
}
```

### 3.2 批量文本处理

后续系统支持 Excel、CSV、TXT 等文件导入，对多条事故警情、投诉文本和新闻舆情进行批量识别，生成结构化表格。

### 3.3 人工校正

系统自动识别结果可能存在缺失或错误，用户可在页面中修改字段。人工校正结果可作为后续优化规则库和训练模型的数据来源。

### 3.4 统计分析展示

系统基于结构化事件数据生成统计图表，包括事件类型分布、高发道路排行、高发时段分析、伤亡情况统计、拥堵等级分布等。

## 4. 系统功能模块

| 模块 | 功能说明 | MVP 是否实现 |
|---|---|---|
| 文本输入模块 | 支持单条中文交通事件文本输入 | 是 |
| 批量导入模块 | 支持 Excel、CSV、TXT 文件批量导入 | 后续 |
| 文本预处理模块 | 文本清洗、分句、时间标准化、地点规范化 | 是 |
| 事件分类模块 | 识别交通事故、拥堵、施工、管制、设施故障等类型 | 是 |
| 实体识别模块 | 抽取时间、地点、车辆、伤亡、道路影响等字段 | 是 |
| 关系抽取模块 | 识别事件与时间、地点、后果、道路影响之间的关系 | 部分 |
| 结构化输出模块 | 输出标准 JSON，后续支持 Excel | 是 |
| 人工校正模块 | 支持对识别结果进行修改与确认 | 后续 |
| 可视化展示模块 | 展示事件统计图表和分析结果 | 后续 |
| 大模型校验模块 | 对低置信度结果进行辅助校验 | 预留 |

## 5. 五层系统架构

```text
数据来源层
事故警情、道路巡查记录、群众投诉、交通新闻、舆情文本、模拟样本
        ↓
文本处理层
文本清洗、去噪、分句、时间标准化、地点规范化
        ↓
智能识别层
规则库、交通领域词典、事件分类、实体识别、关系抽取、大模型辅助校验
        ↓
结构化存储层
JSON 输出、Excel 导出、数据库存储、人工校正记录
        ↓
应用展示层
文本识别、结果展示、人工校正、统计分析、可视化图表
```

## 6. MVP 版本范围

MVP 阶段目标是实现一个可运行的交通事件文本结构化识别原型。

MVP 功能包括：

1. 输入一段中文交通事件文本；
2. 使用规则库和交通领域词典进行识别；
3. 输出 JSON 结构化结果；
4. 支持事件时间、地点、事件类型、涉及车辆、伤亡情况、道路影响、拥堵程度字段；
5. 预留大语言模型接口，便于后续接入校验模块；
6. 提供测试样例，便于演示和验证。

MVP 技术栈：

| 层级 | 技术 |
|---|---|
| 开发语言 | Python |
| 规则识别 | re 正则表达式、自定义词典 |
| 数据格式 | JSON |
| 后续接口 | FastAPI |
| 后续数据库 | SQLite/MySQL |
| 后续前端 | Vue 或 React |
| 后续图表 | ECharts |
