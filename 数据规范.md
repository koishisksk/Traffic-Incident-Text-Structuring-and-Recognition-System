# 《交通事件文本结构化识别系统》数据规范

## 1. 数据规范目标

本规范用于统一交通事件文本结构化识别系统的数据输出格式，服务于规则抽取、模型训练、数据库存储、系统接口开发、统计分析和可视化展示。

系统输入为非结构化中文交通事件文本，输出为标准化交通事件 JSON 数据。

## 2. 交通事件分类体系

采用“一级事件类型 + 二级事件子类型”的分类体系。

| 一级事件类型 | 类型编码 | 二级类型示例 | 说明 |
|---|---|---|---|
| 交通事故 | ACCIDENT | 追尾事故、碰撞事故、剐蹭事故、侧翻事故、单车事故、多车事故 | 车辆、行人或非机动车发生事故 |
| 道路拥堵 | CONGESTION | 通行缓慢、严重拥堵、车辆排队 | 道路通行效率下降 |
| 道路施工 | CONSTRUCTION | 占道施工、道路养护、抢修施工 | 施工、养护或抢修影响道路通行 |
| 交通管制 | CONTROL | 临时封闭、限行、绕行、分流 | 管理部门实施交通限制措施 |
| 设施故障 | FACILITY_FAULT | 信号灯故障、护栏损坏、井盖缺失、标志牌损坏 | 道路交通设施异常 |
| 恶劣天气影响 | WEATHER | 暴雨积水、大雾低能见度、道路结冰 | 天气因素导致交通异常 |
| 违法行为 | VIOLATION | 违停、逆行、占道、闯红灯 | 交通违法造成风险或影响 |
| 其他事件 | OTHER | 无法归类事件 | 兜底类型 |

## 3. 结构化字段表

| 字段名 | 含义 | 类型 | 是否必填 | 示例 |
|---|---|---|---|---|
| event_id | 事件唯一编号 | string | 否 | EVT202607070001 |
| source_text | 原始输入文本 | string | 是 | 今天上午8点30分... |
| source_type | 文本来源类型 | string | 否 | 事故警情 |
| event_type | 一级事件类型 | string | 是 | 交通事故 |
| event_type_code | 一级事件类型编码 | string | 是 | ACCIDENT |
| event_subtype | 二级事件类型 | string/null | 否 | 追尾事故 |
| event_time_text | 原文中的时间表达 | string/null | 否 | 今天上午8点30分 |
| event_time | 标准化事件时间 | string/null | 否 | 2026-07-07T08:30:00+08:00 |
| province | 省级行政区 | string/null | 否 | 重庆市 |
| city | 城市 | string/null | 否 | 重庆市 |
| district | 区县 | string/null | 否 | 南岸区 |
| location | 完整地点描述 | string | 是 | 重庆市南岸区学府大道 |
| road_name | 道路名称 | string/null | 否 | 学府大道 |
| direction | 道路方向或行驶方向 | string/null | 否 | 进城方向 |
| vehicles_involved | 涉及车辆类型 | array | 否 | ["小轿车", "电动车"] |
| vehicle_count | 涉及车辆数量 | integer/null | 否 | 2 |
| casualties | 伤亡原文描述 | string/null | 否 | 1人轻伤 |
| injured_count | 受伤人数 | integer/null | 否 | 1 |
| death_count | 死亡人数 | integer/null | 否 | 0 |
| road_impact | 道路影响描述 | string/null | 否 | 占用右侧车道 |
| occupied_lane | 占用车道 | string/null | 否 | 右侧车道 |
| congestion_level | 拥堵程度 | string | 否 | 缓慢 |
| disposal_measure | 处置措施 | string/null | 否 | 交警已到场处理 |
| event_status | 事件状态 | string | 否 | 处理中 |
| confidence | 识别置信度 | number/null | 否 | 0.92 |
| need_review | 是否需要人工复核 | boolean | 否 | false |
| created_at | 结构化数据生成时间 | string/null | 否 | 2026-07-07T10:00:00+08:00 |

## 4. 字段枚举值

### 4.1 source_type

- 事故警情
- 道路巡查
- 群众投诉
- 交通新闻
- 舆情文本
- 模拟样本
- 其他

### 4.2 congestion_level

- 无明显影响
- 缓慢
- 拥堵
- 严重拥堵
- 未知

### 4.3 event_status

- 待确认
- 发生中
- 处理中
- 已恢复
- 未知

## 6. 数据库表结构

### 6.1 交通事件主表 traffic_event

| 字段名 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增主键 |
| event_id | VARCHAR(64) | 事件编号 |
| source_text | TEXT | 原始文本 |
| source_type | VARCHAR(32) | 来源类型 |
| event_type | VARCHAR(32) | 一级事件类型 |
| event_type_code | VARCHAR(32) | 类型编码 |
| event_subtype | VARCHAR(64) | 子类型 |
| event_time_text | VARCHAR(64) | 原文时间 |
| event_time | DATETIME | 标准化时间 |
| province | VARCHAR(32) | 省级行政区 |
| city | VARCHAR(32) | 城市 |
| district | VARCHAR(32) | 区县 |
| location | VARCHAR(255) | 地点 |
| road_name | VARCHAR(128) | 道路名称 |
| direction | VARCHAR(64) | 方向 |
| vehicles_involved | TEXT | JSON 数组字符串 |
| vehicle_count | INTEGER | 车辆数量 |
| casualties | VARCHAR(128) | 伤亡描述 |
| injured_count | INTEGER | 受伤人数 |
| death_count | INTEGER | 死亡人数 |
| road_impact | VARCHAR(255) | 道路影响 |
| occupied_lane | VARCHAR(128) | 占用车道 |
| congestion_level | VARCHAR(32) | 拥堵程度 |
| disposal_measure | VARCHAR(255) | 处置措施 |
| event_status | VARCHAR(32) | 事件状态 |
| confidence | DECIMAL(4,3) | 置信度 |
| need_review | BOOLEAN | 是否复核 |
| created_at | DATETIME | 创建时间 |

### 6.2 事件实体表 event_entity

| 字段名 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增主键 |
| event_id | VARCHAR(64) | 所属事件编号 |
| entity_type | VARCHAR(32) | 实体类型 |
| entity_text | VARCHAR(255) | 原文实体 |
| normalized_value | VARCHAR(255) | 标准化结果 |
| start_pos | INTEGER | 实体起始位置 |
| end_pos | INTEGER | 实体结束位置 |
| confidence | DECIMAL(4,3) | 置信度 |

### 6.3 事件关系表 event_relation

| 字段名 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增主键 |
| event_id | VARCHAR(64) | 所属事件编号 |
| subject | VARCHAR(128) | 关系主体 |
| relation_type | VARCHAR(64) | 关系类型 |
| object | VARCHAR(255) | 关系客体 |
| confidence | DECIMAL(4,3) | 置信度 |

### 6.4 人工校正记录表 correction_record

| 字段名 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增主键 |
| event_id | VARCHAR(64) | 所属事件编号 |
| field_name | VARCHAR(64) | 被修改字段 |
| old_value | TEXT | 修改前内容 |
| new_value | TEXT | 修改后内容 |
| corrector | VARCHAR(64) | 校正人员 |
| corrected_at | DATETIME | 校正时间 |
